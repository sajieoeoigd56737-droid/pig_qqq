Introduction · 逐句拆解

以 Cyclone IV FPGA 为核心，通过 I2C 总线驱动 AP3216C 三合一光学传感器，实现环境光照度与近距离的实时采集与数码管显示。

Cyclone IV FPGA
相当于整个系统的大脑。它不是普通的芯片——内部逻辑是我们用代码"烧进去"的，负责控制所有时序和数据流。
I2C 总线
只用两根线（时钟线 SCL + 数据线 SDA）就能让 FPGA 和传感器"对话"的通信协议。本项目的 I2C 控制器完全由 Verilog 从零实现。
AP3216C 三合一传感器
一颗芯片同时集成了三种功能：测光亮（ALS）、测距离（PS）、感知红外（IR）。我们用到其中两种：光照度和近距离。
实时采集与数码管显示
FPGA 周期性地把两路数据从传感器读出来，驱动 6 位数码管实时显示。按下按键可切换显示光照值还是距离值。

硬件介绍

野火 FPGA 开发板
搭载 Intel Cyclone IV 系列芯片，50MHz 系统时钟。相当于整个项目的大脑——I2C 通信、状态机采样、数码管驱动，所有逻辑都运行在它上面。
AP3216C 传感器
一颗芯片同时集成了三种传感能力：测光亮度（ALS）、测近距离（PS）、感知红外（IR）。通过 I2C 协议和 FPGA 通信，是整个系统的"眼睛"。
6 位数码管
通过 74HC595 移位寄存器驱动的 6 位共阳数码管，用于实时显示采集到的数值。按下按键可在光照度和距离值之间切换显示。

Data Pipeline · 数据链路

总体框架

采集链路Acquisition
通过 ap3216c_ctrl 调度 i2c_ctrl，经 I2C 总线控制 AP3216C 传感器完成环境光强 ALS 与物体距离 PS 的测量，并读取数据送入 key_ctrl。
ap3216c_ctrl→i2c_ctrl→AP3216C→ALS / PS data
按键交互链路Key Interaction
key_filter 对按键输入进行消抖，输出稳定有效的 key_flag，key_ctrl 根据该信号选择输出 ALS 数据或 PS 数据。
key_in→key_filter→key_flag→key_ctrl
显示链路Display
key_ctrl 输出选中的 20-bit 显示数据，交给 seg_595_dynamic 生成数码管动态扫描时序，将采集结果显示出来。
key_ctrl→data_out[19:0]→seg_595_dynamic→6 位数码管

Module · i2c_ctrl

i2c_ctrl

I2C 主机驱动 把 `ap3216c_ctrl` 给出的读写请求翻译成 AP3216C 的 SCL/SDA 总线时序，完成单字节寄存器写入与单字节寄存器读取。
一次写事务 · wr_en=1，配置寄存器
01
START_1
`i2c_start` 触发后进入起始状态，SCL 高电平期间把 SDA 拉低。
02
SEND_D_ADDR
先发送 7 位设备地址 `0011110`，第 8 位写方向置 0。
03
SEND_B_ADDR_L
`addr_num=0`，只发低 8 位寄存器地址；AP3216C 用 8 位寄存器地址。
04
WR_DATA
把 `wr_data[7:0]` 从高位到低位送上 SDA，例：`0x03` 开启 ALS 与 PS。
05
ACK + STOP
每个字节后释放 SDA 采 ACK；STOP 结束时打一拍 `i2c_end`。
一次读事务 · rd_en=1，寄存器寻址后重复起始
06
Write Address
先按写方向发送设备地址与 `byte_addr`，把传感器内部地址指针定位到目标寄存器。
07
START_2
不释放总线，产生重复起始，再发送设备地址，方向位改为 1。
08
RD_DATA
读取阶段 `sda_en=0`，SDA 变高阻；在 `cnt_i2c_clk=2` 相位采样 8 位数据。
09
N_ACK + STOP
主机不再继续读，所以回 NACK；STOP 后 `rd_data` 锁存并输出完成脉冲。
时钟分频
50MHz `sys_clk` 经 `CNT_CLK_MAX` 分频得到内部 `i2c_clk`，再用 4 个相位控制一个 SCL 周期，输出 250kHz SCL。
位级节拍
`cnt_i2c_clk` 管 SCL 高低相位，`cnt_bit` 管字节内 0~7 位，保证 SDA 在 SCL 低电平换位、高电平稳定。
ACK 判定
进入 ACK 状态时主机释放 SDA，若从机在采样相位拉低 SDA，则 `ack=0`，状态机继续推进。
握手边界
上层只需要给 `wr_en/rd_en` 和一拍 `i2c_start`；底层完成 START、地址、数据、STOP 后用一拍 `i2c_end` 交还控制权。

Module · ap3216c_ctrl

ap3216c_ctrl

AP3216C 采集状态机 负责传感器上电配置、PS/ALS 转换等待、寄存器读请求调度、原始字节拼接和最终采集值输出。
端口信号功能注释 · Inputs / Outputs
i2c_clk
input
I2C 驱动时钟，状态机、等待计数器和输出寄存器都在该时钟下推进。
sys_rst_n
input
低有效复位。清零状态、等待计数、读写控制信号和 PS/ALS 输出缓存。
i2c_end
input
底层 `i2c_ctrl` 的事务完成脉冲。每次读写结束后驱动状态机跳到下一步。
rd_data[7:0]
input
I2C 读回的 8 位数据。按当前状态分别写入 PS 低/高位或 ALS 低/高字节。
wr_en
output
写使能。配置阶段拉高，通知 `i2c_ctrl` 执行寄存器写操作。
rd_en
output
读使能。读取 PS/ALS 寄存器时拉高，通知 `i2c_ctrl` 执行寄存器读操作。
i2c_start
output
启动脉冲。每个读写状态只打一拍，作为底层 I2C 事务的开始命令。
byte_addr[15:0]
output
目标寄存器地址。配置写 `0x00`，PS 读 `0x0E/0x0F`，ALS 读 `0x0C/0x0D`。
wr_data[7:0]
output
写入寄存器的数据。配置 AP3216C 时向 `0x00` 写 `0x03`，打开 ALS 与 PS。
ps_data[9:0]
output
接近传感器结果。由 `0x0E[3:0]` 与 `0x0F[5:0]` 拼成 10 位距离数据。
als_data[15:0]
output
环境光结果。由 `0x0C/0x0D` 拼成 16 位原始值后按 `*35/100` 换算输出。

Display · 显示模块

595 动态扫描

通过 3 线串行接口驱动 74HC595 移位寄存器，控制 6 位共阳数码管进行动态扫描，视觉上呈现稳定清晰的数字显示。

按键消抖计数器 CNT_MAX = 999,999，在 50MHz 系统时钟下对应 20ms 稳定判断窗口，有效消除机械抖动干扰。
